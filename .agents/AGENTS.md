# Workspace Instructions

This workspace is for circular agriculture and agribusiness research, product strategy, and software prototyping.

## Default Behavior

- Treat circular agriculture and agribusiness work as the primary domain.
- Default geography to the Philippines and Southeast Asia when the user does not specify a region.
- Use the `circular-agriculture-operator` skill as the operating mode for research-to-product tasks.
- Use `circular-agriculture-research` first when the task is evidence gathering, opportunity mapping, or literature review.
- Use `circular-agribusiness-app-builder` when the task becomes software, product, dashboard, marketplace, MRV, traceability, logistics, or prototype design.
- Use `deep-research` for broad, comparative, or citation-heavy research.

## Decision Rules

- Prefer ranked opportunities over broad descriptions.
- Tie every recommendation to a user, buyer, workflow, and measurable outcome.
- Separate facts, estimates, and inference.
- Be explicit about confidence, constraints, and evidence gaps.
- Favor the smallest credible workflow or MVP that can be validated in the field.
- Optimize for trust, traceability, operational feasibility, and low-connectivity realities.

## Output Expectations

- For research, end with a concise set of next steps that can support a software product, pilot, or validation plan.
- For product work, define the primary user, buyer, core loop, data model, first screens, and kill criteria.
- If the user’s request is too broad, reduce it to one decision and one best wedge before proposing implementation.
