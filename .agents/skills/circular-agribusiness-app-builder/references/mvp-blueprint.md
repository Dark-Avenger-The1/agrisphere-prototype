# MVP Blueprint

Use this blueprint when turning an idea into a roadmap or prototype plan.

## One-Sentence Product Thesis

For [user segment], this product helps [job] by [mechanism], so they can [measurable outcome].

## MVP Scope

- Primary user:
- Buyer or funder:
- Core workflow:
- Key data inputs:
- Key output:
- Success metric:
- Manual back-office step allowed:
- Excluded from v1:

## First Screens

1. Operational dashboard: what needs attention now.
2. Add record or listing: fastest path to capture useful data.
3. Detail page: evidence, status, history, and next action.
4. Matching, recommendation, or report output.
5. Admin/review queue if verification matters.

## Validation Plan

- Interview 5 to 10 target users about the workflow and current workaround.
- Test a spreadsheet or no-code version before building complex automation.
- Run one mock transaction or pilot batch with real data.
- Measure time saved, revenue created, waste diverted, input cost reduced, or evidence quality improved.
- Define kill criteria before the pilot starts.

## Build Principles

- Build the smallest workflow that produces a real decision, transaction, or report.
- Use domain sample data from the target geography.
- Make quality, uncertainty, and missing data visible.
- Do not overbuild AI features before the data pipeline and user workflow are credible.
