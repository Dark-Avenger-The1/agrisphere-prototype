# Recursive Workflow

Use this loop whenever the user asks for circular agriculture or agribusiness help and the best answer depends on selecting, ranking, or designing something.

## Loop

1. Identify the decision.
   - What must be chosen, built, validated, or compared?
   - Who is the user, buyer, or beneficiary?
   - What geography matters?

2. Find the smallest controlling uncertainty.
   - Is the main question about evidence, economics, operations, policy, or product fit?
   - Do not widen the scope until the controlling uncertainty is named.

3. Gather only enough evidence to decide the next step.
   - Use current sources for market and policy questions.
   - Use deep research if the question is broad or comparative.
   - Prefer ranked evidence over long narrative summaries.

4. Rank the options.
   - Best near-term opportunity.
   - Best software-enablement opportunity.
   - Highest-upside but riskiest opportunity.

5. Recurse if needed.
   - If the ranking is still ambiguous, narrow to one geography, one user segment, or one circular model.
   - If the chosen opportunity is not software-shaped, return to research.
   - If the chosen opportunity is software-shaped, switch to the app-builder skill.

6. End with the smallest credible next action.
   - Research brief.
   - MVP concept.
   - Validation plan.
   - Pilot recommendation.
